General Information


            # What are Operators?
            Operators are the "square-ish" boxes that you drop into your Workflow. You link Operators together so that the output of one Operator flows into the next one. A singular path within the larger Workflow is called a Path. You add/edit the code on the Operator to handle the event.

            # What are Links?
            Links are simply the connections between the Operators. You can also add code to the Link. For example, if the source Operator returns a value of 5, you can add logic on the Link to determine if the Path should continue or stop (e.g. stop processing if the value is less than 4)

            # What are Triggers?
            Triggers are a special kind of Operator that starts your Workflow. For example, the API Ingress Operator is a Trigger that will start the workflow when someone visits the API endpoint. There are also other Triggers such as the Sensor, which is basically a cronjob. You can not create new Triggers.

            # What are Actions?
            Actions are Operators that process the event. For example, you may have a Operator that "Sends a Email" or "Append Row to Google Sheet". Actions can be created and saved for other Workflows. You can NOT create Triggers.

            # What are Lockers?
            Lockers allow you to add Configuration values and Secrets to your Workflow. A Locker can be shared by any number of Workflows. If the Locker is not added to the Workflow, the Workflow can not access it.
